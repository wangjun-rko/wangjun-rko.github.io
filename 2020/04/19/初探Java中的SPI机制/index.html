一、什么是SPI

  • SPI是Service Provider Interface 的简称，即服务提供者接口的意思。
  • 在相应配置文件中定义好某个接口的实现类，然后再根据这个接口去这个配置文件中加载这个实例类并实例化。是一种扩展机制。
  • 哪些地方有用到SPI机制呢？
    • 1、JDBC驱动加载：利用Java的SPI机制，根据不同的数据库厂商来引入不同的JDBC驱动包。
    • 2、SpringBoot：在spring.factories中加上自定义的自动配置类，事件监听器等。

三、源码分析

  • 先来看下ServiceLoader.load(ICar.class)做了什么。
  • LazyIterator是ServiceLoader的内部类，实现了Iterator接口，是一个迭代器。
  • 再来看看serviceLoader.forEach(ICar::getName)做了什么。
    • 执行后会调用ServiceLoader的iterator方法。
    • 返回一个匿名的迭代器对象，重写的hasNext和next方法又分别调用LazyIterator的hasNext和next方法来。
  • 继续执行，进入了LazyIterator的hasNext方法。
  • LazyIterator在用到的时候才会真正去加载实现类。
  • 然后继续执行LazyIterator的next方法。
  • 最后通过返回的实例对象调用其中的方法。

    这里使用了迭代器模式，屏蔽了各个对象的内部差异，提供一个统一的视图来遍历各个对象。

五、线程上下文类加载器

  • java.sql.Driver和java.sql.DriverManager都是在rt.jar包下，由启动类加载器BootstrapClassLoader加载。
  • 数据库驱动类位于classpath中，由于类加载的双亲委派机制，启动类加载器是加载不到这些驱动类的。
  • 如何才能加载到这些驱动类？
    • classpath中的类是由应用程序类加载器AppClassLoader来加载。
    • 在线程里面新定义一个类加载器的属性contextClassLoader。
    • 然后将AppClassLoader设置给contextClassLoader。
    • BootstrapClassLoader去加载java.sql.Driver和java.sql.DriverManager时，同时也会从当前线程中取出contextClassLoader(AppClassLoader)去classpath中加载数据库驱动类。

      通过破坏类加载的双亲委派机制，利用线程上下文类加载器解决了这个问题。
